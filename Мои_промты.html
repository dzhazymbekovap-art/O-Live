Еженедельный отчёт по отделу
Универсальный шаблон — подставляй отдел, период и метрики
ОтчётЕженедельный
Шаблон промта
Ты — аналитик компании O-Live, сервиса доставки правильного питания. Составь еженедельный отчёт для отдела [отдел] за период [период]. Используй следующие метрики: [метрики]. Структура отчёта: 1. Краткое резюме — 2-3 предложения 2. Ключевые показатели в таблице 3. Что шло хорошо 4. Что требует внимания 5. Рекомендации на следующую неделю Стиль: деловой, конкретный, без воды. Данные бери из контекста, который я дам ниже. [вставь данные: цифры, таблицы, выгрузки]
ПеременнаяПримеры значений
[отдел]Доставка / Продажи / Качество / Колл-центр
[период]27 мая 2026 / 20–26 мая / 2-я неделя мая
[метрики]Доставки: число доставок, опоздания, Яндекс-расходы
Продажи: новые клиенты, выручка, средний чек, отток
Качество: жалобы по типам, штрафы, повторные случаи
Колл-центр: новые чаты, подписки, жалобы
[данные]Вставь выгрузку из Bitrix24, Excel или просто цифры
Пример заполнения (колл-центр 27 мая):
[отдел] = Колл-центра · [период] = 27 мая · [метрики] = новые чаты, новые подписки, продажи за день, доставки, жалобы · [данные] = из Bitrix24 и файла Excel

HTML-отчёт по стоимости доставки
KML-зоны + маршрутный лист → HTML с точной стоимостью для каждого адреса
ДоставкиKMLHTML
Шаблон промта
РОЛЬ: Ты — логист компании O-Live, сервиса доставки правильного питания в Алматы. КОНТЕКСТ: Прикрепляю два файла: 1. KML-файл — точные координаты зон доставки (O-live Delivery zone.kml) 2. Excel-файл — маршрутный лист на [дата] (вечер + утро) Правила ценообразования: 🔴 Красная зона («Зона бесплатной доставки» в KML-файле) — 1000 ₸ 🟠 Все адреса вне красной зоны — 1500 ₸ ЗАДАЧА: Для каждого клиента из маршрутного листа: 1. Геокодируй адрес (определи координаты) 2. Проверь, попадает ли он в полигон красной зоны из KML-файла 3. Назначь стоимость: 1000 ₸ (внутри) или 1500 ₸ (снаружи) 4. ОБЯЗАТЕЛЬНО перепроверь каждый адрес через Яндекс Карты с зонами: — Открой: https://yandex.com/maps/162/almaty/?from=mapframe&ll=76.884423%2C43.272655&mode=usermaps&source=mapframe&um=constructor%3Af276b19fa870959f9e2bb0fdf5c0af6f68cb716557c98f123547e49f419c4843&utm_source=mapframe&z=13 — В поиске слева вводи каждый адрес по отдельности и нажимай поиск — На карте будет видно точно, в какой зоне находится адрес — Зона на карте = окончательный ответ (приоритет над геокодом) 5. Если в маршрутном листе есть два одинаковых адреса — удали одну из строк (дубликат не считается отдельной доставкой) ФОРМАТ РЕЗУЛЬТАТА: Создай HTML-отчёт со следующими колонками: № | Имя клиента | Адрес | Зона | Стоимость доставки | Примечание — Раздели на секции: ВЕЧЕР и УТРО — В конце итоговая строка: сколько адресов × 1000 ₸ + сколько × 1500 ₸ = общая сумма — Красная зона — выделить розовым фоном (#FADBD8), внешняя — оранжевым (#FDEBD0) — Заголовок таблицы: тёмный (#2C3E50), шапка колонок: зелёный (#1F7A4D), секции ВЕЧЕР/УТРО: тёмно-синий (#34495E) [прикрепи KML-файл + Excel маршрутный лист]
Результат из файла 24–25 мая: Красная зона: 32 адреса × 1 000 = 32 000 ₸ · Внешняя: 12 адресов × 1 500 = 18 000 ₸ · Итого: 50 000 ₸

ОтзывыКлассификация
Шаблон промта
У меня есть [N] отзывов клиентов в свободном тексте. Для каждого отзыва определи: 1. Тональность: позитив / смешанная / негатив / жёсткий негатив 2. Основная проблема клиента (1 предложение) 3. Темы жалобы (качество, доставка, маркировка, коммуникация и т.д.) 4. Намерение клиента: остаться / пауза / уйти / эскалация 5. Какое действие нужно от компании — срочно или нет Сделай сводную таблицу и в конце выдели, какие отзывы требуют срочной обработки сегодня. [вставь тексты отзывов]
Когда использовать: получила жалобы в WhatsApp, собрала отзывы из Google форм, нужно понять кому ответить первым делом.
Ответ клиенту от отдела качества
Профессиональный ответ на жалобу — с реальными решениями, не отписка
ЖалобаКачество
Шаблон промта
Ты — специалист отдела качества компании O-Live. Клиент [имя клиента] жалуется на: [суть жалобы]. Это [номер] жалоба от этого клиента за последние [период]. Напиши ответ: — Тон: забота, не отписка. Клиент должен почувствовать что его услышали — Признай проблему конкретно, не общими словами — Предложи реальное решение: [компенсация / замена / личный контроль] — Укажи срок ответа или действия — Длина: 5-8 предложений, без воды [опционально: вставь историю переписки с клиентом]
Пример (Тимур, жалоба на ккал): [имя] = Тимур · [жалоба] = несоответствие калорийности на рационе 1800 ккал · [номер] = 4-я жалоба · [решение] = 2 бесплатных дня + личный контроль технолога

3. Юридические документы
Дополнительное соглашение
Шаблон по структуре РОЛЬ → КОНТЕКСТ → ЗАДАЧА → ФОРМАТ
ДоговорРК
Шаблон промта
РОЛЬ: Ты — уполномоченный представитель ТОО «O-Live», ответственный за формирование договорной базы и защиту интересов компании перед поставщиками. КОНТЕКСТ: [Опиши проблему: что происходит, почему нужен документ, кто стороны] ЗАДАЧА: Составь Дополнительное соглашение к договору между [Заказчик] и [Исполнитель] со следующими условиями: 1. [Условие 1 + штраф/ответственность] 2. [Условие 2 + штраф/ответственность] 3. [Условие 3 + штраф/ответственность] ФОРМАТ: — Официально-деловой стиль, структура по законодательству РК — Документ на русском языке, Times New Roman 12pt, формат A4 — Обязательно: преамбула, пронумерованные разделы, таблицы штрафов, порядок претензий (срок ответа — 3 рабочих дня), раздел разрешения споров, блок подписей — Сохрани в формате .docx
Главное правило: Чем подробнее опишешь условия в ЗАДАЧЕ (конкретные суммы штрафов, сроки, основания) — тем точнее будет документ. Не оставляй Claude додумывать цифры.

Шаблон промта
У меня есть скрипт export_whatsapp.py для выгрузки переписок из Bitrix24. Адаптируй скрипт для выгрузки за [дата, например: 2026-05-27]. Жалобы клиентов (негатив, претензии) 2. Вопросы без ответа 3. Запросы на возврат или отмену 4. Сколько всего чатов, сколько с жалобами [прикрепи файл export_whatsapp.py]
